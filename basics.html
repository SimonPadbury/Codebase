Basics

Introduction

Codebase basic styles build upon Normalize (this already imported into codebase.css during Sass pre-processing). These apply to classless HTML tagged elements.

In addition to the basic HTML styling, Codebase has several .text--* utility classes. These have been demonstrated at appripriate points below.

A full list of text utility classes can also be found at Decoration: Text.

Headings

Heading h1

Heading h2

Heading h3

Heading h4

Heading h5
Heading h6

Headings have been sized appropriately for having their line heights fit to a vertical rhythm based on element grid units.

Codebase headings have been set with zero top margin (same as paragraphs).

Headings h1 to h5 are slightly enlarged (and their line heights adjusted for wider viewports) at and above their "alt" breakpoint, which is set by the following variable: 

$break--headers-alt: $mq--sm !default
// = 640px

Headers in Codebase are set to font-weight: normal by default, so that you can use bold on particular words for emphasis.

Also, a set of text utility classes has been provided for modifying any text according to the Codebase headings' font sizes and line heights.

Form Element Sizes

Three sizes have been provided for form inputs, selects and textarea elements. These match the three sizes of buttons.

The smaller form input field size (and smaller button size) has been designed for fitting within menus and menubars.

Form Elements with Big Text

An increased text size of 1.5em has been provided for situations where a form requires special emphasis (e.g. a call to action sign-up).

For the big text size, use the class .form__input--big-text on form elements.

The larger text size is the same as for the optional big text size on buttons.

Big Typography

By default, Codebase has its font sizes for everything based on a set of $text--base-font-* variables. But there is also a set of $text--big-text-font-* variables. (You can override both, because each are set as !default variables.)

There are two ways to deploy big typography in Codebase:

1. The .text--big-text inline element class

If you want to emphasize some inline text by increasing the font size, Codebase can do this by applying .text--big-text to an inline tag (e.g. <span class="text--big-text">).

2. The .text--big-text container element class

Codebase can render big typography simply by placing any Codebase-styled HTML within a container that has class .text--big-text.

This may be useful e.g. for long-read articles.
