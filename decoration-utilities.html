Decoration Utilities

Introduction

For consistency throughout Codebase, several decoration variables have been defined.

These variables have been incoporated into a set of simple decoration utility classes. These classes can be used to quickly add consistent styling to your HTML directly.

You can modify the decoration variables and classes, and add to them, in accordance with the style guide or visual identity of your website project.

Hat tip: The idea for Codebase decoration was inspired by BASCSS, Tachyons and the 8-Point grid.

Notes

  1. If you know you will never use the decoration utility classes, then you can override the $cb-include-css--decoration-utilities to false (see codebase/scss/00_default-variables/_default-variables.scss).
  2. Codebase decoration utility classes are not responsive to viewport sizes.
  3. Beware: there are some circumstances where styling from e.g. your own CSS will override decoration utility classes: CSS nesting. 
/* This CSS will not override decoration utility classes: */

h2 {
  border: 0;
}

<h2 class="b--heavy">Example: this h4 has a heavy border</h2>

Example: this h4 has a heavy border

/* This CSS will override decoration utility classes: */

.infobox h4 {
  border: 0;
}

<div class="infobox">
  <h4 class="b--heavy">Gotcha: this h4 has no border. Why?</h4>
</div>

Gotcha: this h4 has no border. Why?

  1. No styling from anywhere else in codebase.css overrides Codebase decoration utility classes – except the component .decorate classes (but you're not meant to combine .decorate and decoration utility classes anyway).

There is also a .decorate class used for some components. It is used so that the (opinionated) decorative styles can be separated from the functional styles required to set up the components. See Components: Decoration.

Element Grid

Codebase uses a "soft" 4px element grid for consistently positioning elements relative to each other.

This is similar in principle to various 8px grids out there (also known as 8-point or 8dp [device pixel] grids). For example:

  • SpecFM's 8-point grid
  • The BBC GEL's 8px baseline grid
  • Material Design's 8dp square baseline grid

If you're already familiar with the 8-point grid concept, you can think of Codebase's 4px grid in the same way, but with the ability to do a "half unit" whenever you find it useful!

The Codebase element grid has been deployed a follows:

  1. Element grid increments (4px) have been used for setting all line-heights, margins and paddings throughout Codebase.
  2. By default, the base typography vertical rhythm (line-height) is set at 24px (= 3 * 4px, equivalent to line-height: 1.5). The line heights for headers have also been set using 4px increments.
  3. The layout .flexbox gaps have been set at 24px.
  4. Decoration utilities for margins and paddings have been set using values from the 4px element grid.

Margins and Paddings

Codebase decoration utilities for margins and paddings have been set using a $cb-spacing SCSS map, with values 0, 4px, 8px, 12px, 16px, 24px, 48px and 84px.

Margins and paddings utility class prefixes are as follows:

  • Margins all: m--*
  • Margin top: mt--*
  • Margin right: mr--*
  • Margin bottom: mb--*
  • Margin left: ml--*
  • Margins vertical: my--* (top and bottom)
  • Margins horizontal: mx--* (left and right)
  • Paddings all: p--*
  • Padding top: pt--*
  • Padding right: pr--*
  • Padding bottom: pb--*
  • Padding left: pl--*
  • Paddings vertical: py--* (top and bottom)
  • Paddings horizontal: px--* (left and right)

Extra line spacing

Sometimes you may find that you need to add extra whitespace (e.g. above headers, or below list items), equal to the line height of your normal text. Codebase has decoration utility classes for that (for margin and padding). The extra line margins are shown in the following examples.

Margins:

.mt--line
.mb--line

Paddings:

.pt--line
.pb--line

The amount of extra white space applied will be increased if you are also using these classes in combination with big typography.

Block padding

E.g. used for menu and menubar items, tab labels and table cells. Equivalent to py--8 px--16.

.p--block

  • borders all b--*
  • border-top bt--*
  • border-right br--*
  • border-bottom bb--*
  • border-left bl--*

Advanced (SCSS) users:

All Codebase decoration colors are here simply to get you started. You can override and improve them in your own website design as you make progress.

To show you how this is done, notice that there are a few extra example $cb-color--* variables and their instances in the $cb-colors SCSS map in the variables file, commented out.

You need to copy (a.) the whole set of color variables and (b.) the SCSS map into your styles – and then un-comment the extra colors in both. Then during pre-processing, the class-builders in codebase/scss/04_decoration/_colors.scss will generate all the decoration utility classes for colored backgrounds, borders and text.

By overriding the color variables and the SCSS map in this way, you have complete freedom to:

  • Add more color classes
  • Remove classes for colors that you never use
  • Change color class names
  • Change color values

(Outset) Box shadow

Text

Codebase has several simple text utility classes. These are all demonstrated in appropriate places throughout Codebase Basics. But here is the complete listing:

.thin Thin text (default 100)

.bold Bold text (default 700)

.bolder Bolder text (default 900)

.italics Italics text

.normal Returns text to normal weight and removes italicization. Perhaps you may find this useful in some circumstances.

.caps All-capitals text

.muted Muted text is semi-transparent (opacity: 0.5)

.big Big text for making something stand out. (This class works for both inline and block elements.)

.small Small text for small print. (This class works for both inline and block elements.)

.h1 Heading level 1
.h2 Heading level 2
.h3 Heading level 3
.h4 Heading level 4
.h5 Heading level 5
.h6 Heading level 6
