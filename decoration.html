Decoration

Introduction

For consistency throughout Codebase, several decoration variables have been defined. Where necessary, a few of these variables have been used in the various components (applied in .deco classes).

These same variables have been incoporated into a set of simple decoration utility classes. These classes can be used to quickly add consistent styling to your HTML directly.

You can modify the decoration variables and classes, and add to them, in accordance with the style guide or visual identity of your website project.

Hat tip: The idea for Codebase decoration was inspired by BASCSS, Tachyons and the 8-Point grid.

Notes

  1. Codebase decoration classes are not responsive to viewport sizes.
  2. There are some circumstances where styling from e.g. your own CSS will override decoration classes: CSS nesting. 
/* This CSS will not override decoration classes: */

h2 {
  border: 0;
}

<h2 class="b--heavy b--purple">Example: this h4 has a heavy purple border</h2>

Example: this h4 has a heavy purple border

/* This CSS will override decoration classes: */

.infobox h4 {
  border: 0;
}

<div class="infobox">
  <h4 class="b--heavy b--purple">Gotcha: this h4 has no border, why?</h4>
</div>

Gotcha: this h4 has no border, why?

  1. No styling from anywhere else in codebase.css overrides Codebase decoration classes – except the component .deco classes (but you"re not meant to combine deco and decoration classes anyway).

Element Grid

Since version 2.0 Codebase uses a "soft" 4px element grid for consistently positioning elements relative to each other.

This is similar in principle to various 8px grids out there (also known as 8-point or 8dp [device pixel] grids). For example:

  • SpecFM's 8-point grid
  • The BBC GEL's 8px baseline grid
  • Material Design's 8dp square baseline grid

(If you're already familiar with the 8-point grid concept, you can think of Codebase's 4px grid in the same way, but with the ability to do a "half unit" whenever you find it useful.)

The Codebase element grid has been deployed a follows:

  1. Element grid $spacing--* variables have been used for setting all line-heights, margins and paddings throughout Codebase.
Element grid Variable Size
$spacing--4 4px
$spacing--8 8px
$spacing--12 12px
$spacing--16 16px
$spacing--20 20px
$spacing--24 24px
$spacing--28 28px
... ...
$spacing--80 80px
  1. By default, the base typography vertical rhythm (line-height) is set at 24px = $spacing--24 (equivalent to line-height: 1.5). The line heights for headers have also been set using of element grid $spacing--* variables.
  2. The layout .flexbox gutters have been set using $spacing--* variables.
  3. In setting margins and paddings for components in the .deco classes have been set using $spacing--* variables.
  4. Codebase decoraion utilities for margins and paddings have been set using $spacing--* variables.

Margins

Margin utility prefixes:

  • Margins all: m--*
  • Margin top: mt--*
  • Margin right: mr--*
  • Margin bottom: mb--*
  • Margin left: ml--*
  • Margins vertical: mv--* (both top and bottom)
  • Margins horizontal: mh--* (both left and right)

Margin increments as in the following examples (and --zero for removing unwanted margin):

Extra line spacing

Sometimes you may find that you need to add extra whitespace (e.g. above headers, or below list items), equal to the line height of your normal text. Codebase has decoration classes for that (for margin and pading). The extra line margins are shown in the following examples:

.mt--line
.mb--line

The amount of extra white space applied will be increased if you are also using these classes in combination with big typography.

Borders

A few simple border styles.

  • borders all b--*
  • border-top bt--*
  • border-right br--*
  • border-bottom bb--*
  • border-left bl--*

Examples:

.b--zero (0px – for removing borders)
.b--dashed (1px)
.b--thin (1px)
.b--thick (4px)
.b--heavy (8px)

default, borders have color code $color--gray-lighten-2. Borders can be assigned other colors in the Codebase swatch using b--{color} classes. See colors.

Border enhancements:

.b--rounded
.b--circle

Colors

The Codebase color swatches

These colors are here simply to get you started in your prototyping. You can override and add to them in your own website design as you make progress.

Each set of color class builders is individually controlled in the default variables file, so that you can toggle off the classes for colors that you never use.

In each color swatch series there are 7 colors: the base color (see the middle column above) plus 3 incrementally lighter and 3 incrementally darker – plus black and white.
