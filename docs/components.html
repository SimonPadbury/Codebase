Components

Introduction

Codebase components are common HTML/CSS patterns. Some have added functionality from a tiny JavaScript.

Component Mixins and Class Builders

Almost everything in Codebase, including all components, has been created first as a set of Sass mixins. These are then combined and included in their respective class builder mixins. And these class builders are under the control of a set of true !default variables in codebase/sass/01_default-variables/_default-variables.sass.

This means that you can toggle component classes for inclusion (true) or exclusion (false) in your advanced theme build by overriding these defaults.

And this also means that you can include and combine and re-style codebase components in your own theme Sass.

If you do include and use Codebase components in your own theme, then maybe you will not need (some of) the Codebase component classes. For this reason, all Codebase components have their classes under the control of "class builder" toggles, so that you can toggle off unused component classes. This is useful for reducing your final CSS file size.

Also, if you are combining various frameworks with similarly named CSS classes, you can toggle off the Codebase class names, then create new (e.g. prefixed) classnames in order to avoid framework clashes.

Component Deco Styling

Codebase components attempt to be as minimally styled as possible – so that there is not much that you would need to override in your CSS. This can be seen in the "undecorated" examples that follow in this documentation.

However, it is not easy to demonatrate undecorated components. So, for ease of use in the docs some components have some lightweight minimal styling applied by adding the class .deco. You can also use deco styling e.g. during prototyping.

Deco styling consists mainly of adding margin(s) and paddings(s) in accordance with the element grid, and sometimes a thin border, some gray shading (e.g. for back tabs) and box shadows (e.g. for dropdown content).

Therefore, you can add in styling in any of the following ways:

  1. You can, of course, style using your own CSS.
  2. You can use Codebase decoration classes.
  3. You can use .deco on many Codebase components, as listed in the table below.

Component Activator JavaScript

Click a control (e.g. button), and a small JavaScript adds or removes some .is-active classes. That explains the operation of most things in Codebase.

Since Codebase 2, all the various showhides, dropdowns, menubars, popovers, and off-canvas sidebars are controlled by one script called _activator.js, which has been pre-processed and included within codebase.js.

Basic activator components need an outer container element (block), with an inner control element and a content element beside it (as siblings). They can be anything, so long as the controller has a class with suffix __ctrl and the content has a class with suffix __content.

Example:

<div class="thing">

  <button class="thing__ctrl">
    Click me
  </button>
  
  <div class="thing__content">
    I am a thing.
  </div>
  
</div>

It doesn't matter whether the controller element comes before or after the content element.

class="thing"
class="thing__ctrl"
class="thing__content"
class="thing"
class="thing__content"
class="thing__ctrl"

You can utilize the activator script. To do so, give your component's outer HTML container a class name, and then use that same class name as a prefix for its controller and its content.

Activator Operation

It is entirely up to you what styling you wish to add for the inactive and active states of the component, controller and content. E.g. Codebase uses CSS transitions for the animation of showhide, collapsed menubar, off-canvas sidebar and popover.

  1. Click/tap an activator control .thing__ctrl to add class .is-active to all these:
    1. Its .thing block
    2. The clicked .thing__ctrl element itself
    3. Its respective .thing__content element
  2. At the same time, all other activator components will get .is-active removed. Therefore, only one activator component can be active at any time.
  3. Click/tap an activator .thing__ctrl again to cause the removal of the .is-active classes, thereby deactivating this component.
  4. A click/tap anywhere outside an activator content will also deactivate the active component (useful for dismissing an off-canvas sidebar or a popover/ modal).
  5. A click/tap an on-page <a href="#..."> tag within an activator will also deactivate the active component (useful for dismissing an off-canvas sidebar if visiting a hyperlink target on the same webpage).
  6. Optional extra: click/tap a .thing__close element within an active activator component to deactivate it. (See "Closing from within", below.)
  7. Optional extra: adding the modifier .thing--no-dismiss to the component block will stop propapgation of a click/tap through the block, thereby preventing "click/tap outside to dismiss" (useful for situations when you don't want a visitor to dismiss a popover/ modal without interacting with it).

Examples of simple activator components in Codebase are dropdown and showhide.

Adding specificity

Add data-content-id="identifier" to control elements in order to target particular id="identifier" content elements.

This enables the controller to be located outside the component (e.g. for use in creating popovers of off-canvas sidebars).

<div class="thing">

  <div id="thing-content-2" class="thing__content">
    I am thing 2.
  </div>

</div>

<button data-content-id="thing-content-2" class="thing__ctrl">
  Click me
</button>

Closing from within

If for some reason you wish to have a "close" (or "dismiss") button inside your content element, you can do that. Simply add an element with class thing__close inside your thing__content. Clicking this element will cause the removal of is-active from the content element (and the control element, if you have one).

Codebase popovers and off-canvas sidebars have "close" buttons.

Menubar: Responsive

There may be situations where you require a navigation list to display as a stacked menu on small viewports but as a menubar on wider viewports.

Codebase handles this using mixins/classes menu--menubar-*, where * can be xs, sm, md or lg for transformation at any of the various breakpoints in Codebase.

Note: the responsive menu / menubar mixins and classes are named thinking "from small screens up". They are menus that become menubars above the chosen breakpoint.

Incorporating dropdown menus

When a dropdown is incorporated into a menubar, the first click on a parent link activates the drodown to reveal the child menu items (without the parent link being followed).

Clicking the parent link again (while the dropdown content is activated) will follow the parent link to its linked page.

Clicking anywhere else outside the activated dropdown content will dismiss the dropdown. (Clicking another dropdown will deactivate the active dropdown and activate the newly clicked dropdown instead.)

Note: you won't have to use the responsive dropdown modifier classes (e.g. .dropdown--md-up) because the responsive menubar modifier class (e.g. .menu--menubar-md) incorporates dropdown mixins above the breakpoint width.

Popover

A Codebase popover is what is sometimes known as a modal in other frameworks. It is also sometimes used as a lightbox (if the main content is an image). These are essentially all the same thing, therefore "popover" has been chosen as the most generic name.

The .popover component block itself serves at the popover backdrop.

If you add the modifier class .popover--no-dismiss to the popover block, then "click/tap outside to dismiss" is prevented. This can be useful in situations where your popover content is rather important, and you need to ensure that your visitor interacts with it before deactivating the popover.

Also, popovers can be closed by a .popover__close element situated incise the popover content element.

Popovers "expand to fit" any size content

If you don't use any popover constraint classes (as described below), then you get a popover – that will take the width of its content, out to the max-width of the viewport and 85% of the viewport height. (Example 1, above, has its max-width as wide as the content – i.e. the "Lorem ipsum" sentance.) The viewport height restriction allows for e.g. the iOS Safari browser bar.

You can add margin and padding, to suit your design – e.g. using Codebase decoration utility classes.

By default, popovers have their .popover__content have CSS overflow: auto, so that extra long text and/or table content will scroll.

Images that are too tall for the popover content will be scaled down to max-height 70% of the viewport height (image width/height proportions are retained). This max-height ensures that images can be entirely displayed within the popover and also allow a little room for an image title. (This provides a very simple "lightbox" setup.)

Styling your own modals and lightboxes

If you need your popovers, lightboxes and modals to diverge and be styled differently, there are two ways you can do this:

  1. Add your own supplemental class(es) and apply your styles to these (e.g. .popover--modal).
  2. In your Sass, build your own classes, include the popover mixins and then add your own styling. E.g. 
.your-lightbox
  +popover
  // and your additional lightbox styles

.your-lightbox__ctrl
  +popover__ctrl
  // and your additional lightbox styles

.your-lightbox__content
  +popover__content
  // and your additional lightbox styles
  &--narrow
    +popover--narrow
  &--cover
    +popover--cover

.your-lightbox__close
  +popover__close
  // and your additional lightbox styles

Popover Classes

The table below includes only classes for the popover itself. For card classes see card.

Class Explanation
.popover The popover block. Also serves as the popover backdrop.
.popover--no-dismiss (Optional) Prevents "click/tap putside to dismiss" happening through the backdrop. So the only way to close the popover would be by .popover__close (see below).
.popover__ctrl
data-content-id="{ID}"		 Opening control (e.g. button)
{ID} = the ID of the popover content to be opened
Note: Do not put the .popover__ctrl inside the .popover container, because .popover is styled for use as the backdrop.
.popover__close (Optional) popover close control, to be placed within the popover content element.
.popover__content The popover itself. (A unique ID is also required, of your choice – see data-content-id="{ID}" above.)
By default the popover content width is unconstrained – so the width becomes whatever the content needs ("expand to fit").
.popover__content--narrow (Optional) narrow constrained layout
.popover__content--fix (Optional) if a .card is being used in a popover, this class will fix .card__header and/or .card__footer while the .card__body remains vertically scrollable. Ideal for use with constrained-width popovers that have a lot of content.
.popover__content--cover (Optional) full-page cover layout

A tab label's data-tab-id="" refers to its respective card's id="".

Note: Codebase automatically sets the first tab to be in front (i.e. on display) when the page is loaded.

HTML
Pug
<div class="tabs">

  <div class="tab__label" data-tab-id="tabs-example-1">Tab Label 1</div>
  <div class="tab__label" data-tab-id="tabs-example-2">Tab Label 2</div>
  
  <div class="tab__card" id="tabs-example-1">
    <p>Tab Card 1</p>
  </div>
  
  <div class="tab__card" id="tabs-example-2">
    <p>Tab Card 2</p>
  </div>

</div>

.tabs
  .tab__label(data-tab-id="tabs-example-1")
  .tab__label(data-tab-id="tabs-example-2")

  .tab__card#tabs-example-1
    p Tab Card 1

  .tab__card#tabs-example-2
    p Tab Card 2

Tab labels can either each be placed next to their respective content (as in the example code above), or they can be separate – e.g. all the labels grouped together first in the tabs container.

X-Axis Scrolling on Tab Labels

If you group all the tab labels together as above, you have the option to wrap them in a container with class tab__label-wrap. This container prevents the tabs from wrapping to a newline (when there are more tabs than the viewport width allows on row of labels), and allows x-axis scrolling.
