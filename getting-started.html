Getting Started

Introduction

Codebase is a website component library. You can use it as the basis for your website design projects.

The Codebase stylesheet codebase.css has been composed using indented syntax Sass. Its styles were built first as mixins and then these have been included in the Codebase classes.

Codebase styles can therefore be used simply as-is, by dropping the Codebase classes directly into your website HTML – or whatever you use to compile your HTML (e.g. pug). Alternatively, you can include Codebase mixins into your own Sass.

A few things in Codebase also use a tiny JavaScript codebase.js to make them work – such as dropdowns, popovers and responsive tables. Since version 2.0, Codebase scripts have been re-factored as "vanilla" JavaScript and so jQuery is not needed.

The Codebase documentation (you are looking at it) is an example of Codebase in use.

Besides these docs, some Codebase webpage examples have been included in this project.

Simple Setup

  1. Add codebase.css and codebase.js to your project.
  2. Create your first HTML page – e.g. using the starter example below. 
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1">
    <meta name="viewport" content="width=device-width, initial-scale=1">

    <title>Your Webpage Title</title>

    <link rel="stylesheet" href="path/to/codebase/css/codebase.css">
    <link rel="stylesheet" href="path/to/your-theme/css/your-styles.css">
  </head>
  <body>

    <h1>Get to work</h1>

    <script src="path/to/codebase/js/codebase.js"></script>
    <script src="path/to/your-theme/js/your-scripts.js"></script>
  </body>
</html>
  1. Build your HTML web page prototype(s) following the examples in these docs.
  2. Personalize with your own stylesheet, scripts and graphics.

Third-party Dependencies

codebase.css has CSS @includes that source the following from CDNJS while you are online (they may already be in your browser cache):

  1. The Normalize stylesheet.
  2. The Font Awesome stylesheet and icon font.

│   └── sass/ [2]
│
├── your-build/
│   ├── css/
│   ├── img/
│   └── js/
│
└── your HTML

Notes:

  1. The Codebase documentation has been built using the HTML pre-processor language Pug (formerly known as Jade). You can do the same. See the further instructions below on how to use Codebase with Pug.
  2. The Codebase CSS has been built using indented syntax Sass. You can do the same. See the further instructions below on how to use Codebase with Sass.

The Codebase CSS and JavaScript components can, of course, be used directly on the HTML files (as in the Simple Setup, above). Or you can use e.g. PHP, Handlebars, Slim, or Haml.

Using Codebase with Sass

You can @import the Codebase Sass master file directly into your project Sass master file, and then all the Codebase variables and mixins will be available for you to use. E.g.: 

@import "../../codebase/scss/codebase"

All Codebase variables have the !default flag. To override any of these, you must first declare your own overrides using the same Codebase variable names (but with your preferred values), and then import Codebase. 

// (1.) Override Codebase `!default` variables before importing Codebase

$color--yellow: #ffff00

// (2.) Import Codebase

@import "../../codebase/scss/codebase"

// (3.) Begin your project styles after importing Codebase

body
  background: $color--yellow

Your Sass pre-processor will preferentially use your values instead of the Codebase defaults. You won't need to modify any Codebase Sass file directly.

You can copy the entire Codebase default variables file (codebase/scss/02_default-variables/_default-variables.scss) into your theme, then remove all the !default flags, and set your chosen variable values there.

Media Query Breakpoint Widths

In Codebase the media query breakpoint width variables are:

$mq--xs: 320px !default
$mq--sm: 640px !default
$mq--md: 960px !default
$mq--lg: 1280px !default

These have been utilized in a set of Sass mixins as follows:

=break--min($mq--min-width)
  @media(min-width: $mq--min-width)
    @content

=break--max($mq--max-width)
  @media(max-width: $mq--max-width - 1px)
    @content

=break--minmax($mq--min-width, $mq--max-width)
  @media(min-width: $mq--min-width) and (max-width: $mq--max-width - 1px)
    @content

These mixins are then called as in the following examples:

// For styling only applicable below `$mq--sm` (i.e. up to 639px)

+break--max($mq--sm)
  ...

// For styling only applicable from `$mq--sm` to `$mq--md` (i.e.  640px to 959px)

+break--minmax($mq--sm, $mq--md)
  ...

// For styling only applicable above `$mq--md` (i.e. from 960px and up)

+break--min($mq--md)
  ...

Using Codebase with Pug

The Codebase documentation has been developed using PugJS. The Pug language was (and is being) developed by Forbes Lindesay.

One way of using Codebase with Pug in your project is as follows: Codebase comes with a simple starter Pug template called _codebase.pug, which you can inherit (using extends) as the basis of your templates. (Alternatively, you can start without extending this example starter template).

The _codebase.pug starter template

//- Codebase Starter

include generators/__generators-index

block vars
  - lang = ""
  - htmlDir = ""
  - htmlClass = ""
  - title = ""
  - description = ""
  - author = ""
  - favicon = ""
  - faviconType = ""
  - bodyClass = ""

doctype html
html(lang=lang, dir=htmlDir, class=htmlClass)

  head
    meta(charset="utf-8")
    meta(http-equiv="X-UA-Compatible", content="IE=edge, chrome-1")
    meta(name="viewport", content="width=device-width, initial-scale=1, shrink-to-fit=no")
    meta(name="description", content=description)
    meta(name="author", content=author)

    title #{title}

    block head-addins
      link(rel="icon", href=favicon, type=faviconType)
      link(rel="stylesheet", href="codebase/css/codebase.css")

  body(class=bodyClass)
    block body

    block body-foot-addins
      script(src="codebase/js/codebase.js")

You can then extend the Codebase starter in your own Pug template(s) as follows: 

//- Your Template

extends ../codebase/pug/_codebase

//- ...etc.

How Codebase uses Pug

Pug is a whitespace-sensitive (indented syntax) templating language for pre-processing into HTML. With Pug you can use partial-page includes, loops, conditionals, inheritance, mixins and variables.

The PugJS documentation is the first place to learn Pug.

Plus there are several tutorials out there. Following are some to get you started. (Pug was formerly known as Jade, and some of the following tuturials date from that time but are still relevant.)

  • Simon Swiss's series, Jade/Pug Tips in 90 Seconds on YouTube
  • Derek Banas's Jade/Pug Tutorial on YouTube
  • Bryan Stoner's tutorial, Learn Pug.js with Pugs on Codepen

The Codebase docs were generated using many of the features of Pug. So, you can learn from the docs code itself, as well as reading the docs in a browser.

  1. The docs have been mostly written using the Pug syntax. (Some HTML is also embedded.)
  2. The Codebase docs have a their own docs starter template named _docs-template.pug, that itself interits the Codebase starter template named _codebase.pug. (It is this Codebase starter template that links to codebase.css and codebase.js.)
  3. The docs starter template has blocks into which the docs chapter pages (e.g. getting-started.pug) insert their content – see (4.) and (5.).
  4. The docs-template-partials/ folder contains partial Pug files (e.g. header, sidebars, footer and article templates) that are included into the docs pages.
  5. Each chapter article has its own article template (_docs-article.pug).
  6. The docs-articles/ folder contains all the articles of the Codebase docs. These articles are partial files that are included into the pug pages, similar to how the header, sidebars, etc. are included.
  7. The docs make use of some simple Codebase HTML generator mixins that are used for generating some of the docs examples. They can also be used for rapid prototyping (see their use in the Examples).

Codebase HTML Generator Mixins

Codebase has some handy Pug HTML generator mixins. These have been used in generating some of the docs and examples, and you can also use them for rapid prototyping.

Following these example HTML generators, you can also generate your own. Examples of others in the the Codebase project can be found in the .popover--fix example (a popover with a lot of tabular content), and in the example blog index excerpts loop.

"Lorem ipsum" Paragraph Generator

When this paragraph mixin is included, it will generate up to four paragraphs in your HTML depending on how many you specify. You can also add some classes to the first paragraph (e.g. if you require it to have a dropcap, or enlarged "lead" text, or some decoration classes).

See codebase/pug/generators/_paragraphs.pug. 

mixin paragraph(number, p1Class)
  case number
    when 1
      p(class=p1Class).
        Lorem ipsum dolor sit amet ...
    when 2
      p(class=p1Class).
        Lorem ipsum dolor sit amet ...
      p.
        Summo nostro dicunt sea ut ...
    when 3
      p(class=p1Class).
        Lorem ipsum dolor sit amet ...
      p.
        Summo nostro dicunt sea ut ...
      p.
        Utroque habemus moderatius cum cu ...
    when 4
      p(class=p1Class).
        Lorem ipsum dolor sit amet ...
      p.
        Summo nostro dicunt sea ut ...
      p.
        Utroque habemus moderatius cum cu ...
      p.
        Eruditi omittam usu et ...
    default
      Lorem ipsum dolor sit amet ...

(In the actual mixin these paragraphs are written in full.)

Usage:

//- To get one paragraph (default):

+paragraph

//- To get a specific number of paragraphs (up to 4)

+paragraph(2)

//- To add some classes to the first paragraph

+paragraph(1, "b--heavy")

+paragraph(3, "lead text--purple")

Menu and Menubar Generators

Mixins for rapid generation of menus and menubars have been provided.

See codebase/pug/generators/_menus.pug. 

//- Menu generator

mixin menu(menuLink, total, menuClass)
  - var n = 0;
  ul(class=["menu", menuClass])
    while n < total
      li
        a(href="#/")= menuLink
          = " "
          = (n++ + 1)

//- Menubar generator

mixin menubar(menuLink, total, menuClass)
  - var n = 0;
  ul(class=["menubar", menuClass])
    while n < total
      li
        a(href="#/")= menuLink
          = " "
          = (n++ + 1)

Usage: in the parameters specify:

  1. The text of the menu link
  2. The number of menu items
  3. Any classes you require to be added (optional)

Examples: 

//- a 5-item menu:

+menu("Menu link", 5)

//- a 4-item menubar, centered

+menubar("Test", 4, "menubar--center")

Table Generator

This mixin builds a demo table for you.

See codebase/pug/generators/_tables.pug. 

mixin table(totalColumns, totalRows, tableClass)
  table(class=["table", tableClass])
    thead
      tr
        - for (var cols = 0; cols < totalColumns; cols++)
          th= "Table Header"
            = " "
            = (cols + 1)
    tbody
      - for (var rows = 0; rows < totalRows; rows++)
        tr
          - for (var cols = 0; cols < totalColumns; cols++)
            td= "Table content"
              = " "
              = (cols + 1)
              = "."
              = (rows + 1)

Usage: in the parameters specify:

  1. The number of rows
  2. The number of columns
  3. Any classes you require to be added (optional)

Examples: 

+table(3, 4)

+table(6, 5, "bg--yellow")

RTL Languages Support

Codebase aims to support RTL languages without the requirement for special RTL classes to be applied to elements or containers of elements. Codebase does this by using the CSS3 [dir="rtl"] selector.

Add dir="rtl" to a container – this can be the <html> tag itself. This sets the direction of the text flow from right to left – and it implements Codebase RTL support on those components that need it.

If for some reason you really can't use [dir="rtl"], then you can use class="rtl" – either applied to the element itself, or to a container of it. (But of course, classes do not change the text direction. So you will still have to accomplish that by another means.)

If you add [dir="rtl"] (or class="rtl") to a container, component or element that requires it, then you get Codebase RTL support.

Some Codebase components and text utility classes have a *--right and *--left modifier class for controlling the horizontal alignment of elements. These have been styled so that they are the same for both LTR and RTL. (Right is always right and left is always left.)

Codebase was originally built by an LTR language speaker (English), and so the Sass has been styled from a "LTR first" perspective, with RTL overrides built into its mixins – as in the highlighed selectors in the following simplified example (Sass): 

=something

  // LTR supported by default
  padding-left: 100px
    
    // RTL supported by override
    [dir="rtl"] &, &[dir="rtl"],
    .rtl &, &.rtl
      padding-left: inherit
      padding-right: 100px

.something
  +something
