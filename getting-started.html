Getting Started

Introduction

Codebase is a website component library. You can use it as a basis for your website design projects.

The Codebase stylesheet codebase.css has been composed using SCSS. Its styles were built first as mixins and then these have been included in the Codebase classes.

Codebase styles can therefore be used simply as-is, by dropping the Codebase classes directly into your website HTML – or whatever you use to compile your HTML (e.g. pug). Alternatively, you can include Codebase mixins into your own SCSS.

The Codebase documentation is an example of Codebase in use.

A few things in Codebase also use a simple, tiny JavaScript codebase.js to make them work – such as dropdowns, popovers, tabs and responsive tables. Since version 2.0, Codebase scripts have been re-factored so as not to need a dependency – jQuery is not required.

Besides these docs, some Codebase webpage examples have been included in this project.

Simple Setup

  1. Add codebase.css and codebase.js to your project.
  2. Create your first HTML page – e.g. using the starter example below. 
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1">
    <meta name="viewport" content="width=device-width, initial-scale=1">

    <title>Your Webpage Title</title>
    
    <link rel="stylesheet" href="path/to/codebase/css/codebase.css">
    <link rel="stylesheet" href="path/to/your-theme/css/your-styles.css">
  </head>
  <body>

    <h1>Get to work</h1>

    <script src="path/to/codebase/js/codebase.js"></script>
    <script src="path/to/your-theme/js/your-scripts.js"></script>
  </body>
</html>
  1. Build your HTML web page prototype(s) following the examples in these docs.
  2. Personalize with your own stylesheet, scripts and graphics.

Example project folder structure:

your-project/
│
├── codebase/
│   ├── css/ [1]
│   ├── js/
│   ├── pug/ [2]
│   └── scss/ [3]
│
├── your-dev/
│   ├── img/
│   ├── js/
│   ├── pug/ [2]
│   └── scss/ [3]
│
└── your-build/
    ├── css/
    ├── img/
    ├── js/
    └── HTML

Notes:

  1. If you are using the Codebase styles as a Sass (SCSS) mixin library, then you will be including codebase.scss in your own (master) stylesheet and so the Codebase CSS will not be required in your project.
  2. The Codebase CSS and JavaScript components can, of course, be used directly on the HTML files. Or you can use HTML templating languages with as Pug, PHP, Handlebars, Slim, Liquid or Haml. The Codebase documentation has been built using Pug. If you would like to do the same, see using Codebase with Pug.
  3. The Codebase CSS has been built using SCSS. You can override Codebase variables and incorporate Codebase variables and mixins into your own SCSS. See using Codebase with SCSS.

Using Codebase as a Sass (SCSS) Library

Codebase is a highly versatile Sass (SCSS syntax) library, built using variables and mixins. You can import the Codebase SCSS master file directly into your project SCSS master file, and then all the Codebase variables and mixins will be available for you to use.

@import "../../codebase/scss/codebase"

Note:

  • All Codebase variables are prefixed $cb-
  • All Codebase mixins are prefixed cb-

Using Codebase Variables

All Codebase variables have the !default flag. To override any of these, you must first declare your own overrides using the same Codebase variable names (but with your preferred values), and then import Codebase. 

// (1.) Override Codebase `!default` variables before importing Codebase

$cb-color--blue: #000099;

// (2.) Import Codebase

@import "../../codebase/scss/codebase";

// (3.) Begin your project styles after importing Codebase

body {
  // This will now get `your` $cb-color--blue, not the !default in Codebase
  background: $cb-color--blue;
}

Your SCSS pre-processor will preferentially use your values instead of the Codebase defaults. You don't need to modify any Codebase SCSS file directly.

Tip: You can copy the entire Codebase default variables file (codebase/scss/00_default-variables/_default-variables.scss) into your theme, then remove all the !default flags, and set your chosen variable values there.

Using Codebase Mixins

In Codebase most* of the styles have been built first as mixins. And then these have been included in the Codebase CSS classes.

(* i.e. all styles that use CSS classes and some basic styles that apply to HTML tags).

At the bottom of the Codebase variables file, you will find a set of $cb-include-css-* variables. 

// Basics
$cb-include-css--typography:             true !default;
$cb-include-css--floats-clearfix:        true !default;
$cb-include-css--buttons:                true !default;
$cb-include-css--badges:                 true !default;
$cb-include-css--labels:                 true !default;
$cb-include-css--combos:                 true !default;
$cb-include-css--forms:                  true !default;

// Layout
$cb-include-css--container:              true !default;
$cb-include-css--flexbox-system:         true !default;
$cb-include-css--invisibility:           true !default;

// Components
$cb-include-css--cards:                  true !default;
$cb-include-css--dropdowns:              true !default;
$cb-include-css--menus-and-menubars:     true !default;
$cb-include-css--off-canvas:             true !default;
$cb-include-css--pagination:             true !default;
$cb-include-css--popovers:               true !default;
$cb-include-css--showhide:               true !default;
$cb-include-css--tables:                 true !default;
$cb-include-css--tabs:                   true !default;

// Decoration
$cb-include-css--decoration-utilities:   true !default;

You can:

  • Choose whether or not to include these classes by toggling the following `true/false` variables (i.e. by overriding them in your own variables). Toggle `false` whatever you don't need, for generating smaller CSS. Never use popovers (modals or lightboxes)? Then don't include them.
  • Include these mixins in your own classes (with your own choice of class names). This may be useful if you are combining various frameworks and you need to have components etc. differently namespaced to avoid clashes.
  • Include Codebase mixins at different media query breakpoints.
  • Switch all the Codebase CSS classes off, and then use only your own classes.

While the Codebase docs focus on demonstrating the Codebase CSS classes and JS functionality, Sass/SCSS users will still find the docs useful because the mixin names and class names are essentially the same. And the docs navigation organization follows the SCSS folder/file organization, so familiarization with the docs can help you quickly find your way around the SCSS.

Media Query Breakpoint Widths

In Codebase the media query breakpoint width variables are:

// (Small is up to 639px)

// Medium
$cb-mq--md: 640px !default;

// Large
$cb-mq--lg: 960px !default;

Also note: the max width of the content inside a layout container is 1280px. (Containers also have 12px padding (i.e. 3 element grid units) right and left, beyond the 1280px.)

These media query widths are intended to cover a wide range of screen devices, approximating as follows:

Small Viewports Medium Viewports (md) Large Viewports (lg)
0 639px 640px 959px 960px max. 1280px
← phones (portrait) → ← phones (landscape) →
← tablets (portrait) →		 ← tablets (landscape) & up →

The media query breakpoint variables been utilized in a set of SCSS mixins as follows:

@mixin break--min($cb-mq--min-width) {
  @media(min-width: $cb-mq--min-width) {
    @content;
  }
}

@mixin break--max($cb-mq--max-width) {
  @media(max-width: $cb-mq--max-width - 1px) {
    @content;
  }
}

@mixin break--minmax($cb-mq--min-width, $cb-mq--max-width) {
  @media(min-width: $cb-mq--min-width) and (max-width: $cb-mq--max-width - 1px) {
    @content;
  }
}

These mixins are then called as in the following examples:

// For styling only intended for small viewports

@include break--max($cb-mq--md) {
  ...
}

// For styling only intended for medium viewports

@include break--minmax($cb-mq--md, $cb-mq--ld) {
  ...
}

// For styling only intended for large viewports

@include break--min($cb-mq--lg) {
  ...
}

Using Codebase with Pug

The Codebase documentation has been developed using PugJS. The Pug HTML-templating language is being developed by Forbes Lindesay on GitHub.

Pug is a whitespace-sensitive (indented syntax) templating language for pre-processing into HTML. With Pug you can use partial-page includes, loops, conditionals, inheritance, mixins and variables.

One way of using Codebase with Pug in your project is as follows: Codebase comes with a simple starter Pug template called _codebase.pug, which you can inherit (using extends) as the basis of your templates. These docs and examples begin that way.

Alternatively, you can start without extending this example starter template.

The PugJS documentation is the first place to learn Pug.

Plus here are some tutorials:

  • Simon Swiss's series, Jade/Pug Tips in 90 Seconds on YouTube
  • Derek Banas's Jade/Pug Tutorial on YouTube
  • Bryan Stoner's tutorial, Learn Pug.js with Pugs on Codepen

Note: Pug was formerly known as Jade, and some of the tutorials above date from that time but they are still relevant.

The _codebase.pug starter template

//- Codebase Starter

include generators/__generators-index

block vars
  - lang = ""
  - htmlDir = ""
  - htmlClass = ""
  - title = ""
  - description = ""
  - author = ""
  - favicon = ""
  - faviconType = ""
  - bodyClass = ""

doctype html
html(lang=lang, dir=htmlDir, class=htmlClass)

  head
    meta(charset="utf-8")
    meta(http-equiv="X-UA-Compatible", content="IE=edge, chrome-1")
    meta(name="viewport", content="width=device-width, initial-scale=1, shrink-to-fit=no")
    meta(name="description", content=description)
    meta(name="author", content=author)

    title #{title}
    
    link(rel="icon", href=favicon, type=faviconType)

    block head-addins

      //- If you wish to replace `codebase.css` with your own CSS (that imports 
        `codebase.css`), then use `block` in your template. Or if you wish to retain this `codebase.css` below, then use `block append head-addins`.
      link(rel="stylesheet", href="codebase/css/codebase.css")
        
  body(class=bodyClass)
    block body

    block body-foot-addins
      script(src="codebase/js/codebase.js")

You can then extend the Codebase starter in your own Pug template(s) as follows: 

//- Your Template

extends ../codebase/pug/_codebase

//- ...etc.

How the Codebase Docs use Pug

The Codebase docs were generated using many of the features of Pug. So, you can also learn from the docs code itself.

  1. The docs have been written using the Pug syntax.
  2. The Codebase docs have a their own docs starter template named _docs-template.pug, that itself inherits the Codebase starter template named _codebase.pug. (It is this Codebase starter template that links to codebase.css and codebase.js.)
  3. The docs starter template has blocks into which the docs chapter pages (e.g. getting-started.pug) insert their content – see (4.) and (5.).
  4. The docs-template-partials/ folder contains partial Pug files (e.g. header, sidebars, footer and article templates) that are included into the docs pages.
  5. Each chapter article has its own article template (_docs-article.pug).
  6. The docs-articles/ folder contains all the articles of the Codebase docs. These articles are partial files that are included into the pug pages, similar to how the header, sidebars, etc. are included.
  7. The docs make use of some simple Codebase HTML generator mixins that are used for generating some of the docs examples. They can also be used for rapid prototyping (see their use in the Examples).

Pug-HTML Generator Mixins

Codebase has some handy Pug-HTML generator mixins for rapid prototyping.

These have been used in generating some of the docs examples and example templates.

You can use these generators and also develop your own if you need them in your prototyping. Examples of others in the Codebase project can be found in the .popover--fix example (a popover with a lot of demo tabular content), and in the example demo blog excerpts loop.

"Lorem ipsum" Paragraph Generator

When this paragraph mixin is included, it will generate one to five paragraphs in your HTML depending on how many you specify. You can also add some classes to the first paragraph (e.g. if you require it to have a dropcap, or enlarged "lead" text, or some decoration classes).

See codebase/pug/generators/_paragraphs.pug.

//- PARAGRAPH GENERATOR

mixin paragraph(number, p1Class)
  case number
    when 1
      p(class=p1Class) Lorem ipsum dolor sit amet...
    
    when 2
      p(class=p1Class) Lorem ipsum dolor sit amet...
      p Cras bibendum libero non vulputate congue...
    
    when 3
      p(class=p1Class) Lorem ipsum dolor sit amet...
      p Cras bibendum libero non vulputate congue...
      p Cras vel massa nec magna tincidunt maximus a in erat...
    
    when 4
      p(class=p1Class) Lorem ipsum dolor sit amet...
      p Cras bibendum libero non vulputate congue...
      p Cras vel massa nec magna tincidunt maximus a in erat...
      p Aliquam erat volutpat. Maecenas pulvinar tincidunt augue...
    
    when 5
      p(class=p1Class) Lorem ipsum dolor sit amet...
      p Cras bibendum libero non vulputate congue...
      p Cras vel massa nec magna tincidunt maximus a in erat...
      p Aliquam erat volutpat. Maecenas pulvinar tincidunt augue...
      p Fusce pretium eros sit amet ante porta, a ullamcorper elit vulputate...

    default
      p Lorem ipsum dolor sit amet, consectetur adipiscing elit...

(In the actual mixin these paragraphs are written in full.)

Usage examples:

//- To get one paragraph (default):

+paragraph

//- To get a specific number of paragraphs (up to 4)

+paragraph(2)

//- To add some classes to the *first* paragraph

+paragraph(1, "dropcap gray")

+paragraph(3, "lead blue-darken-1")

Menu and Menubar Generators

Mixins for rapid generation of demo menus and menubars have been provided. See codebase/pug/generators/_menus.pug.

//- Menu generator

mixin menu(menuLink, total, menuClass)
  - var n = 0;
  ul(class=["menu", menuClass])
    while n < total
      li
        a(href="#/")= menuLink
          = " "
          = (n++ + 1)

//- Menubar generator

mixin menubar(menuLink, total, menuClass)
  - var n = 0;
  ul(class=["menubar", menuClass])
    while n < total
      li
        a(href="#/")= menuLink
          = " "
          = (n++ + 1)

Usage: in the parameters specify:

  1. The text of the menu link
  2. The number of menu items
  3. Any classes you require to be added (optional)

Examples:

//- a 5-item menu:

+menu("Menu link", 5)

//- a 4-item menubar, centered

+menubar("Test", 4, "menubar--center")

Table Generator

This mixin builds a demo table for you.

See codebase/pug/generators/_tables.pug.

mixin table(totalColumns, totalRows, tableClass)
  table(class=["table", tableClass])
    thead
      tr
        - for (var cols = 0; cols < totalColumns; cols++)
          th= "Table Header"
            = " "
            = (cols + 1)
    tbody
      - for (var rows = 0; rows < totalRows; rows++)
        tr
          - for (var cols = 0; cols < totalColumns; cols++)
            td= "Table content"
              = " "
              = (cols + 1)
              = "."
              = (rows + 1)

Usage: in the parameters specify:

  1. The number of rows
  2. The number of columns
  3. Any classes you require to be added (optional)

Examples:

+table(3, 4)

+table(6, 5, "table--cols table--striped")

Image Placeholder Generators (SVG)

Codebase has its own rapid SVG placeholder generators. These simply have a width and height, a background color and a "X" composed of diagonal lines joining opposite corners. A light version and a dark version have been provided – chooses whatever is appropriate for your prototype design.

Alternatively you may use e.g. http://holderjs.com or https://placeholder.com.

Examples:

+image-placeholder-light(100, 100)

+image-placeholder-dark(100, 100, 'b--circle')

  - var imageWidth
  - var imageHeight  
  
  svg(xmlns="http://www.w3.org/2000/svg" preserveAspectRatio="xMinYMin meet" width=imageWidth height=imageHeight class=imageClass style="max-width: 100%; height: auto; background: #eee")

RTL Languages Support

Codebase aims to support RTL languages without the requirement for special RTL classes to be applied to elements or containers of elements. Codebase does this by using the CSS3 [dir="rtl"] selector.

Add dir="rtl" to a container – this can be the <html> tag itself. This sets the direction of the text flow from right to left – and it implements Codebase RTL support on those components that need it.

If you add dir="rtl" (or class="rtl") to a container, component or element that requires it, then you get Codebase RTL support.

If for some reason you really can't use dir="rtl", then you can use class="rtl" – either applied to the element itself, or to a container of it. (But of course, classes do not change the text direction. So you will still have to accomplish that by another means.)

Some Codebase components and text utility classes have a *--right and *--left modifier class for controlling the horizontal alignment of elements. These have been styled so that they are the same for both LTR and RTL. (Right is always right and left is always left.)

Codebase was originally built by an LTR language speaker (English), and so the SCSS has been styled from a "LTR first" perspective, with RTL overrides built into its mixins – as in the highlighed selectors in the following simple example (SCSS): 

@mixin something {

  // LTR supported by default
  padding-left: 100px;
    
  // RTL supported by override
  [dir="rtl"] &, &[dir="rtl"],
  .rtl &, &.rtl {
    padding-left: inherit;
    padding-right: 100px;
  }
}
