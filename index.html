Dependencies

None. The compiled Codebase CSS and JS are all that you need.

codebase.css already has Normalize built in. You don't need to add it separately.

codebase.js has been totally rebuilt and simplified for v.2.0. And now it does not ueed jQuery (or any other library).

No icon fonts are required by Codebase itself – you can use any icon font you choose. (These docs and examples use Font Awesome).

Two ways of using Codebase

  1. Simple – go ahead and use codebase.css and codebase.js in your project.
  2. Advanced – use Codebase as a Sass (SCSS) mixin library, and do what you want with its JS.

You can also use the Codebase Pug starter template, that was developed for this documentation. But Codebase CSS and JS works just fine with HTML or any templating language.

The Codebase CSS

Codebase styling (in both the SCSS and the docs) is organised into Basics and Components (analogous to Atomic Design's "Atoms" and "Molecules").

All Codebase variables are set with !default values – so, they can be easily overridden. And if you @import the Codebase master SCSS file codebase.scss into your own SCSS master file, then you can (override and) utilize and the variables, and you can include and/or extend the mixins in your own project.

Most* Codebase styles has been created first as SCSS mixins and then these are included in classes – all following BEM naming convention.
(* i.e. all styles that use CSS classes and some basic styles that apply to (unstyled) HTML tags)

Find out more about using Codebase with SCSS.

The Codebase JS

A few things in Codebase (responsive tables, showhide, dropdown, off-canvas, popovers and tabs) use some tiny JavaScripts. These are combined into one, codebase.js.

Codebase Layout Systems

Codebase has a versatile flexbox 'columnar grid' system – traditional 12 columns (default), responsive, with column span, offset, reordering, centering and wrapping.

A range of invisibility mixins/classes have also been included.

Decoration Utilities

Codebase components are as 'cosmetically unstyled' as possible. This has been done so that there is not much that you would need to override in your CSS.

However, some minimal decorative styling can be applied by adding a .decorate class to each component individually where you wish to use it. (And when you don't want Codebase opinionated styling, in the SCSS variables file you can toggle off the .decorate classes.)

Codebase also has some simple decoration utility classes – which are similar in principle to BASCSS, Tachyons, and the utilities in the new Bootstrap 4.

All margins, paddings, font sizes and line heights are built on a 4px element grid, similar in principle to Spec FM's 8-Point Grid.

Codebase decorative styling uses a common set of decoration variables – which you can easily override.

Using Codebase with Pug

The Codebase CSS and JS files can be used with HTML directly, of course. Or you can use a templating language such as PugJS.

The Codebase documentation has been developed using Pug. Pug snippets of Codebase components have been included through the docs (look for the Pug tabs, such as in the "Get started" button example below). There are also further examples.

Find out more about using Codebase with Pug.

Browser Compatibility

Codebase has been tested and it all works with current and recent versions of Safari (MacOS and iOS), Firefox, Chrome, Opera, Vivaldi, Brave and Edge.

Codebase is also mostly OK on IE10 and IE11 (only these don't work well: popovers, card with fixed header/scrolling body, and SVG images within flexbox layout.) These few problems have to do with old IE's incomplete implemetation of flexbox.
