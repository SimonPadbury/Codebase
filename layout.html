Layout

Flexbox Systems

Since Codebase 2.0, the flexbox layout mixins and classes formerly using the word "grid" have been renamed simply as layout flexbox and "col" (column) has been renamed as flexitem. This has been done so that the word "grid" can instead be used in the names of mixins and classes to refer to the new CSS grid specification.

Mixin (prefix) Class (prefix)
The flexbox block +flexbox .flexbox
The flex-item element (immediate child) +flexitem .flexitem

In the Codebase flexbox system, all the following options are supported:

  • Flexitem span
  • Flexitem offseting
  • Flexitem (re)ordering
  • Flexitem centering (horizontally) and/or middling (vertically)

Incomplete flexitem "rows"? No problem. Flexbox "rows" can be incomplete. The last flexitem does not expand to occupy the unused space. No additional "end" classes required.

Surplus horizontal flexitems ("columns")? No problem. Flexitems will automatically wrap if they extend beyond the flexbox container max width. The last flexitem does not expand to occupy the unused space. No additional "end" classes required.

Flexboxes at different Media Query Breakpoints

The Codebase media query breakpoint widths are specified by four !default variables.

Flexboxes can be used with or without specifying breakpoints (and with or without using breakpoint classes). Without breakpoints, the flexitems will be displayed the same (% width) whatever the window width.

This means that each flexbox system can have up to five viewport width configurations:

.flexitem classes +flexitem mixins

Flexbox Variables

Codebase flexbox systems use the following variables:

Variable Value Explanation
$mq--xs 320px !default for extra-small viewports (e.g. phones, portrait).
$mq--sm 640px !default for small viewports (e.g. phones, landscape; tablets, portrait).
$mq--md 960px !default Break-point for medium viewports (e.g. tablets, landscape).
$mq--lg 1280px !default Break-point for large viewports (e.g. laptops, desktops).
$max-flexitems 12 !default The default Codebase flexbox system has up to 12 columns (flexitems), each 1/12 of the flexbox max width.
$flexbox--max-width 1280px !default Maximum width of the modifiers +flexbox--container mixin and .flexbox--container class. For wider viewports the flexbox is centered by auto-margins right and left.
$flexbox--gutter-width $spacing--24 !default of the gutter width is used as margin either side of each flexbox column. (See element grid for units.)

Class Explanation
.flexbox The container class. Sets up the flexbox. Its width is unconstrained.
.flexbox--container The .flexbox--container block modifier constrains the flexbox to $flexbox--max-width and, on wider viewports, centers the flexbox by auto-margins right and left.
.flexbox--gutter Gives right and left padding to all immediate child block elements (i.e. flexitems).
.flexbox--nested Gives negative right and left margin to the flexbox container equal to the gutter width, so that a nested flexbox is flush right and left inside their containing flexitem.
.flexbox--center Applied to the flexbox container, this will horizontally center any number of flexitems.
.flexbox--middle Applied to the flexbox container, this will vertically center any number of flexitems.
.flexitem The immediate child ("column" or "row") base class. It has a fallback to width: 100% for causing a flexitem to become full-width when the viewport is below the lowest flexitem span breakpoint.

Notes on flexitem (re-)ordering:

  1. If you require column re-ordering, specify the order of all flexitems in your flexbox.
  2. If at a higher breakpoint you need columns to revert to their normal order (as in the HTML), then you can use --order-0] (e.g. .flexitem-lg--order-0).

The Codebase semantic flexbox and flexitem mixins are named similar to the Codebase classic flexbox and flexitem classes. However, note that there are no breakpoints built into these mixins – you can apply them in your Sass as in the example above.

Mixin Explanation
+flexbox The container mixin. Sets the max width (from variable $flexbox--max-width), does centering by auto margin right and left. If you have more columns (or column width) than the flexbox container width, wrapping is handled by flex-wrap: wrap;

If a flexbox container doesn't have the flexbox mixin included, then a flexbox system will expand to whatever the width of the container is, without maxing out at $flexbox--max-width. E.g. it could expand to the <body> width.
+flexbox--container +flexbox--container block modifier constrains the flexbox to $flexbox--max-width and, on wider viewports, centers the flexbox by auto-margins right and left.
+flexbox--gutter Gives right and left padding to all immediate child block elements (flexitems).
+flexbox--nested Gives negative right and left margin to the flexbox container equal to the gutter width, so that a nested flexbox is flush right and left inside their containing column.
+flexbox--center to the flexbox container, this will horizontally center any number of flexitems.
+flexbox--middle to the flexbox container, this will vertically center any number of flexitems.
+flexitem The flexitem base mixin.
+flexitem--span(X) Column spanning using % widths (note: not using flex-grow), as calculated out of $max-flexitems (default is 12).
+flexitem--offset(X) offsetting (at different breakpoints), as calculated out of max $max-flexitems (default is 12).
+flexitem--order(X) Column (re-)ordering.

Notes

  1. There are no Codebase .show-- classes. Therefore, if you need something to be visible only at some viewport widths and not others, then what you need to do is use the appropriate .hide--* class so that it will not be visible at viewport widths where you don't want it to be.
  2. If you need something to be visible only at a mid-range viewport width, then use two .hide--* classes: one to hide the thing at lower viewport widths and one to hide it at higher viewport widths. E.g. 
// A div that is ONLY visible at medium viewport widths:

<div class="hide--md-down hide--lg-up"> ... </div>

For an example of where this has been done, see the viewport indicator (fixed to the bottom-right corner) in these docs.
