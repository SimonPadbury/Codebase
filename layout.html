Layout

Layout Container

If you need your design layout to have a maximum width, you can use the Codebase .container class.

<div class="container"> ... </div>

The container class controls the block maximum width to 1280px, and it centers the container auto-margins right and left.

You can use a container to wrap a flexbox layout system.

<div class="container">
    <div class="flexbox"> ... </div>
</div>

Flexbox Columnar Grid System – Introduction

Since Codebase 2.0, the flexbox layout mixins and classes formerly using the word "grid" have been renamed simply as layout flexbox and "col" (column) has been renamed as flex-item*.

(So, the classname "grid" is free to be used on a proper CSS grid system in future.)

Mixin (prefix) Class (prefix)
The flexbox block @include flexbox(--*) .flexbox(--*)
The flex-item element (immediate child) @include flex-item--* .flex-item--*
*Note: In the Codebase CSS there is no styling for the class .flex-item – so you don't actually need to use this class anywhere. The simple flexbox doesn't need classes on the immediate child elements (i.e. the flex-items) at all. But .flex-item-- is used as a prefix for the traditional flexbox (responsive) span, offset, order and center classes.

The Codebase flexbox system can be used in two ways:

  1. Simple flexbox – .flexbox simply puts the immediate child elements in a row. They will each occupy their own required (but there are some modifier classes that can be used for "growing" flex-items). Simple flexbox does not respond to media query breakpoints.
  2. Traditional flexbox – this supports all the following options:
    • Flex-item span
    • Flex-item offset
    • Flex-item (re)order
    • Flex-item center (horizontally) and/or middle (vertically)
    • Incomplete flexbox "rows" – the last flex-item does not expand to occupy the unused space, and no additional "end" class is required
    • Flex-item wrapping – surplus flex-items will automatically wrap if they extend beyond the flexbox container max width
    • Responsive flexbox layouts at different media query breakpoint widths

Flexbox and Flex-Item Classes

Class Explanation
Flexbox classes
.flexbox The container class. Sets up the flexbox. Its width is unconstrained.
.flexbox--grow-equal Flexbox modifier class – makes (immediate child) flex-items to grow to equal width.
.flexbox--grow-auto Flexbox modifier class – makes (immediate child) flex-items to grow to equal width.
.flexbox--container Flexbox modifier class – constrains the flexbox to $cb-flexbox--max-width and, on wider viewports, centers the flexbox by auto-margins right and left.
.flexbox--gap Gives right and left padding to all immediate child block elements (i.e. flex-items).
.flexbox--center Applied to the flexbox container, this will horizontally center any number of flex-items.
.flexbox--middle Applied to the flexbox container, this will vertically center any number of flex-items.
Flex-item classes
.flex-item--*
.flex-item-md--*
.flex-item-lg--*		 Controlling the column (flex-item) width behaviour at different breakpoints. Except for .flex-item--* all flex-items expand to width: 100% of their container width below their breakpoint.
*--span-* Flex-item spanning at different breakpoints using % widths (note: not using flex-grow), as calculated out of max $cb-max-flex-items (default is 12).
*--offset-* Flex-item offsetting (at different breakpoints), as calculated out of max $cb-max-flex-items (default is 12).
*--order-* Flex-item (re-)ordering (at different breakpoints).

Flexbox (Traditional) at different Media Query Breakpoint Widths

The Codebase media query breakpoint widths are specified by four !default variables.

Flexboxes can be used with or without specifying breakpoints (and with or without using breakpoint classes). Without breakpoints, the flex-items will be displayed the same (% width) whatever the window width.

This means that each flexbox system can have up to five viewport width configurations:

Flex-Item classes Flex-Item mixins
1 Multi-column at all screen widths .flex-item--* Use flex-item mixins without wrapping them in breakpoint mixins.
2 Multi-column at or above $cb-mq--md (default 640px) .flex-item-md--* Wrap flex-item mixins in
@include break--min($cb-mq--md) {...}
3 Multi-column at or above $cb-mq--lg (default 960px) .flex-item-lg--* Wrap flex-item mixins in
@include break--min($cb-mq--lg) {...}

Flexbox Variables

Codebase flexbox systems use the following variables:

Variable Value Explanation
$cb-mq--md 640px !default Break-point for medium viewports (e.g. tablets, landscape).
$cb-mq--lg 960px !default Break-point for large viewports (e.g. laptops, desktops).
$cb-max-flex-items 12 !default The default Codebase flexbox system has up to 12 columns (flex-items), each 1/12 of the flexbox max width.
$cb-flexbox--gap-width $cb-spacing--24 !default of the gap width is used as margin either side of each flexbox column. (See element grid for units.)

Using Flexbox Mixins for Semantic Layouts

Most Codebase flexbox layout mixins are used for generating the Codebase flexbox layout clases. As such, they are not suitable for Sass (SCSS) users to include in their code. But Codebase also has a semantic flexbox layout system for Sass users.

See:

  1. codebase/scss/02_layout/_shared-flexbox-base.scss.
  2. codebase/scss/02_layout/_semantic-flexbox-system.scss.

The Codebase semantic flexbox and flex-item mixins are named similar to the Codebase classic flexbox and flex-item classes. However, note that there are no breakpoints built into these mixins – you can apply them in your SCSS as in the example above.

Mixin Explanation
Flexbox mixins
@include flexbox The container mixin. Sets the max width (from variable $cb-flexbox--max-width), does centering by auto margin right and left. If you have more columns (or column width) than the flexbox container width, wrapping is handled by flex-wrap: wrap;

If a flexbox container doesn't have the flexbox mixin included, then a flexbox system will expand to whatever the width of the container is, without maxing out at $cb-flexbox--max-width. E.g. it could expand to the <body> width.
@include flexbox--grow-equal Flexbox modifier mixin – makes (immediate child) flex-items to grow to equal width.
@include flexbox--grow-auto Flexbox modifier mixin – makes (immediate child) flex-items to grow to equal width.
@include flexbox--container Flexbox modifier mixin – constrains the flexbox to $cb-flexbox--max-width and, on wider viewports, centers the flexbox by auto-margins right and left.
@include flexbox--gap Gives right and left padding to all immediate child block elements (flex-items).
@include flexbox--center to the flexbox container, this will horizontally center any number of flex-items.
@include flexbox--middle Applied to the flexbox container, this will vertically center any number of flex-items.
Flex-item mixins
@include flex-item--span(X) Column spanning using % widths (note: not using flex-grow), as calculated out of $cb-max-flex-items (default is 12).
@include flex-item--offset(X) Flexitem offsetting (at different breakpoints), as calculated out of max $cb-max-flex-items (default is 12).
@include flex-item--order(X) Column (re-)ordering.

  1. There are no Codebase .show-- classes. Therefore, if you need something to be visible only at some viewport widths and not others, then what you need to do is use the appropriate .hide--* class so that it will not be visible at viewport widths where you don't want it to be.
  2. If you need something to be visible only at a mid-range viewport width, then use two .hide--* classes: one to hide the thing at lower viewport widths and one to hide it at higher viewport widths. E.g.
// A div that is ONLY visible at medium viewport widths:

<div class="hide--md-down hide--lg-up"> ... </div>

For an example of where this has been done, see the viewport width indicator (fixed to the bottom-right corner) in these docs.
